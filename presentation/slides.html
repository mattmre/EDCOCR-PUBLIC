02 / 16

The Problem

Most OCR pipelines fail at the questions that matter for legal, medical, and regulated work.

  • Generative models hallucinate text that was never on the page.
  • "Good enough" OCR silently drops images, low-confidence runs, and unreadable scans.
  • There's rarely a trustworthy log of which page got which model, at which DPI, when.
  • Language detection is one-shot — multilingual documents come out garbled.
  • Cloud APIs are non-starters for SCIF, HIPAA-adjacent, or classified environments.
03 / 16

The Approach

Engineered around three load-bearing decisions.

CTC Decoding

PaddleOCR with CTC heads. No generative tokens, no synthesized characters. The recognizer can refuse, but it cannot invent.

Hash-Chained Custody

Every document, page, and transform is recorded in a SHA-256 hash-chained JSONL log. Tampering one entry breaks the chain.

Forensic Preservation

OCR failure never deletes the page. Failed pages embed the rasterized original so nothing is lost between source and deliverable.

Air-Gap Ready

All models pre-baked into Docker images. No runtime calls to external services. Bundle scripts produce SCIF-deployable artifacts.

06 / 16

45 Languages, Adaptive

Two-pass detection

  • FastText runs first on a quick sample.
  • Best-matching PaddleOCR model is loaded.
  • Detection re-runs on extracted text to catch mixed-script pages.

Tiered registry

  • Core (34): EN, FR, DE, ES, IT, PT, NL, RU, AR, FA, HI, CH, JP, KO, and 20 more.
  • Extended (+11): TH, BN, MR, NE, HR, SK, NO, LT, LV, ET, RS_LATIN.
  • Toggle via OCR_LANGUAGE_TIERS=core,extended.

Per-span language detection is opt-in (ENABLE_PER_SPAN_LANGUAGE=true). Emits a .language.json sidecar with BCP-47 tags per line, page, and document.

Deployment Topologies

Same image. Three places to run it.

Single Machine

docker compose up

GPU or CPU. Watch a folder, drop documents in, results land in ocr_output/. Designed to fit on a workstation.

Distributed

helm install edcocr

Django coordinator + Celery + Postgres + RabbitMQ + Redis. KEDA autoscaling on queue depth. Per-GPU queue affinity.

Air-Gapped

./airgap-bundle.sh

Pre-baked models, bundled images, zero outbound. Tar -> sneakernet -> airgap-deploy.sh on the isolated side.

11 / 16

Security Posture

  • Non-root containers. All 6 Dockerfiles run as UID 1000, capabilities dropped, seccomp default.
  • Per-tenant isolation. Tenant ID flows through every API, DB filter, and output path.
  • API key auth + rate limiting. X-API-Key header, slowapi limits per endpoint.
  • Webhook secrets encrypted at rest. Fernet, key derived from WEBHOOK_SECRET_KEY.
  • SSRF protection. Outbound URL fetches are allowlisted; metadata IPs blocked.
  • Audit logging. Hash-chained custody log; tamper-evident.
  • Subprocess timeouts. Ghostscript 300s, Tesseract 120s, Poppler 300s.

14 / 16

Translation (Opt-In)

Pluggable engine adapters with license-aware routing.

Engines

  • OPUS-MT (Apache 2.0) — default
  • NLLB-200 (CC-BY-NC-4.0) — opt-in
  • MADLAD-400 (Apache 2.0)
  • Generative LLM tier (Ollama, vLLM, OpenAI-compatible) — opt-in

Policy

  • Per-tenant allow_nc_licensed flag
  • Privilege-blocked content never routes to cloud generative
  • Custody event on engine selection + every translation
  • Certification gated behind strong-auth review

All translation flags default OFF. ENABLE_TRANSLATION=true to opt in.
