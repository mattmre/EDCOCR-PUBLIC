# Code of Conduct

EDCOCR follows the **Contributor Covenant, version 2.1**, the most widely adopted code of conduct in open source.

The full text is maintained upstream and is available at:

- HTML: <https://www.contributor-covenant.org/version/2/1/code_of_conduct/>
- Markdown: <https://www.contributor-covenant.org/version/2/1/code_of_conduct/code_of_conduct.md>

By participating in this project — opening issues, submitting pull requests, posting in Discussions, or interacting with other contributors — you agree to abide by that document in full.

## What it means in this project

In short:

- **Be civil and be technical.** Disagreements about design are welcome; personal attacks are not.
- **Assume good faith.** Other contributors usually want the same thing you do, even when their proposed approach is different.
- **Respect time.** People reviewing your PR are doing it on their own clock. Be concise, be specific, and follow up on review comments rather than re-litigating them.
- **Inclusion is non-negotiable.** Everyone who shows up to contribute is welcome regardless of background, identity, employer, or experience level.

## Reporting

If you experience or witness behavior that you believe violates this code of conduct, **do not post about it publicly**. Use the private channel:

- Send a private report via [GitHub Security Advisories](https://github.com/mattmre/EDCOCR-PUBLIC/security/advisories/new) (the same channel used for vulnerability disclosures — both go to maintainers only).
- Or send a direct message to [**@mattmre**](https://github.com/mattmre) on GitHub.

Reports are read by the maintainers only and are kept confidential. Retaliation against good-faith reporters is itself a violation of this code of conduct.

## Enforcement

Maintainers will follow the **enforcement guidelines** described in §4 of the Contributor Covenant 2.1 (Correction → Warning → Temporary Ban → Permanent Ban), with the specific response calibrated to the severity and pattern of the behavior.

## Scope

This code of conduct applies in all project spaces — the repository, Issues, Pull Requests, Discussions, any project-related chat — and also to behavior in other venues when an individual is representing the project or its community.
