Why EDCOCR?

Most OCR tools were built for digitizing brochures. EDCOCR was built for the day when someone asks "where did this text come from, and can you prove it?"

[CTC]

Zero Hallucination

PaddleOCR with CTC decoding — no generative models, no fabricated tokens. What's on the page is what gets extracted.

[CHAIN]

Chain of Custody

Every document, every page, every transform recorded in a SHA-256 hash-chained JSONL audit log. Tamper-evident by design.

[SAFE]

Forensic Preservation

OCR failure never discards evidence. Failed pages embed the original image so the document is always recoverable.

[GPU]

GPU Accelerated

6-stage async pipeline, 31 concurrent threads, adaptive batch sizing. Tuned for sustained throughput on 16-48GB VRAM cards.

[LANG]

45 Languages

FastText language detection drives per-page model swapping. Latin, Cyrillic, CJK, Arabic, Indic, and Southeast Asian scripts.

[AIR]

Air-Gap Ready

Pre-baked Docker images, bundled models, no runtime network calls. Designed for SCIF, regulated, and disconnected environments.

Quick Start

Pick a deployment path. All three produce identical OCR output.

Docker

Single-machine, GPU or CPU, watch a folder.

terminal 
git clone https://github.com/\
mattmre/EDCOCR-PUBLIC.git
cd EDCOCR-PUBLIC
./scripts/docker-quickstart.sh

Kubernetes

Production cluster, Helm, KEDA autoscaling.

terminal 
helm install edcocr \
  helm/ocr-local/ \
  -f values-secret.yaml \
  --namespace edcocr \
  --create-namespace

Python SDK

Talk to a running cluster from your app.

python 
from edcocr_sdk import EDCOCRClient

c = EDCOCRClient(
  base_url="...",
  api_key="..."
)
job = c.submit_job("contract.pdf")

Designed For

Legal Discovery & eDiscovery

Bates stamping. Redaction-aware exports. Privilege-keyword detection. Tamper-evident audit log every reviewer can verify. CTC-only OCR keeps extracted text defensible.

Healthcare & Regulated Records

PII/PHI spatial extraction with bounding-box fidelity. Per-tenant isolation. Air-gap deployment for HIPAA-adjacent workflows. Custody trail satisfies common compliance asks.

Government & Public Records

FOIA backlogs. Multi-language scans. Handwriting detection. Signature verification (advisory). Designed for environments where generative AI is contractually off the table.

Enterprise Document Operations

Classification, entity recall, structured extraction, review queue, webhook delivery, Prometheus metrics. The boring infrastructure plumbing you only notice when it's not there.

What Comes Out

Every job emits a structured set of outputs. Pick what your downstream needs.

Output Format What it contains
EXPORT/PDF/ PDF Searchable PDF with embedded OCR text layer
EXPORT/TEXT/ .txt Plain text extraction
EXPORT/STRUCTURE/ JSON Layout regions, table cells (HTML), figure positions
EXPORT/NER/ JSON Named entities including CASE_NUMBER and BATES_NUMBER
EXPORT/CLASSIFICATION/ JSON Document type with confidence
EXPORT/EXTRACTION/ JSON Structured fields — dates, amounts, names, addresses
EXPORT/HANDWRITING/ JSON Handwriting regions with confidence
EXPORT/LANGUAGE/ JSON Per-span language detection with BCP-47 tags
EXPORT/VALIDATION/ JSON Per-page confidence + quality classification
EXPORT/RETRIEVAL/ JSON + Markdown Unified retrieval format for downstream search
